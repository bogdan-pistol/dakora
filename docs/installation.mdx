## Requirements

- Python 3.11 or higher
- pip or uv package manager

<Info>
  For the interactive playground:
  - PyPI releases include a pre-built UI (no additional setup)
  - Development installs require Node.js 18+ (builds automatically on first run)
</Info>

## Install from PyPI

<CodeGroup>

```bash pip
pip install dakora
```

```bash uv
uv add dakora
```

</CodeGroup>

## Development Installation

For contributing or development:

```bash
git clone https://github.com/dakora-labs/dakora.git
cd dakora
uv sync
source .venv/bin/activate
```

<Tabs>
  <Tab title="macOS/Linux">
    ```bash
    source .venv/bin/activate
    ```
  </Tab>
  <Tab title="Windows">
    ```bash
    .venv\Scripts\activate
    ```
  </Tab>
</Tabs>

## Verify Installation

```bash
dakora --version
```

You should see the version number printed.

## Authentication Setup (Optional)

To enable authentication with Clerk:

### Frontend

Create `studio/.env.local`:

```bash
# Enable authentication
VITE_AUTH_REQUIRED=true

# Get from Clerk Dashboard: https://dashboard.clerk.com
VITE_CLERK_PUBLISHABLE_KEY=pk_test_YOUR_KEY_HERE

# Backend API
VITE_API_URL=http://localhost:8000
```

For development without authentication:

```bash
VITE_AUTH_REQUIRED=false
```

### Backend

Set environment variables for authentication:

```bash
# Enable authentication
AUTH_REQUIRED=true

# Get from Clerk Dashboard: https://dashboard.clerk.com â†’ API Keys
CLERK_JWT_ISSUER=https://your-domain.clerk.accounts.prod.liveblocks.io
CLERK_JWKS_URL=https://your-domain.clerk.accounts.prod.liveblocks.io/.well-known/jwks.json

# Database (required for authenticated requests)
DATABASE_URL=postgresql://user:password@localhost:5432/dakora
```

For development without authentication:

```bash
AUTH_REQUIRED=false
```

See the [Authentication Guide](/guides/authentication) for detailed setup instructions.

## Optional Dependencies

### For LLM Execution

Dakora uses LiteLLM for LLM execution. Install provider-specific SDKs if needed:

<CodeGroup>

```bash OpenAI
pip install openai
```

```bash Anthropic
pip install anthropic
```

```bash Google
pip install google-generativeai
```

</CodeGroup>

<Note>
  Most providers work out-of-the-box with LiteLLM. Provider SDKs are only needed for specific features or local models.
</Note>

### For Environment Variables

To use `.env` files for API keys:

```bash
pip install python-dotenv
```
